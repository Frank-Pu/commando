## 维度 1：产品 / 服务

- **一句话定义**：LeMingle 是一个帮助外语学习者**自然记住地道表达、提升词汇量**的浏览器插件。核心机制：网页浏览中**自动高亮地道搭配**+ 显示翻译解释。
- **当前阶段**：**探索 PMF 中**
  - 注册用户：~100
  - 付费用户：**个位数**
  - MAU：难统计
- **历史增长尝试**：小红书 + 知乎 内容营销 + SEO
- **产品发心**：Qihang 自己留学经历（海外做 AI 研究，做 presentation / 参会交流时"看过的专业词说不出来"）→ 因此设计了"日常任务过程中顺带学英语"的产品逻辑

## 维度 2：真实 ICP

- **当前付费用户画像（n=几个）**：海外高净值高素质专业人士
  - 案例 1：用户 A（在美国服务跨语言专业场景）
  - 案例 2：用户 B（海外华人研究员）
- **Qihang 自评**：实际付费 ICP **与最初设想基本一致**——海外学习/工作中需要专业英语的人
- **增长痛点**：如何**精准触及**这批人
- 🚨 **Onboarding 风险记录**：n=几个 不足以确认 ICP。Playbook 警示——用户答"ICP 和我想的一致"时要警惕 confirmation bias

## 维度 3：北极星 + 红线

- **CAC 红线**：倾向自然增长 + 内容营销；不排斥按 LTV 反推投流红线

## 🚨 ICP 描述的修正（关键认知更新）

**之前 Onboarding 误判**：以为 ICP 是"海外律师/PhD"，于是判断小红书不匹配。
**真信号**：用户 A 案例说明 ICP **是"海外华人专业人士"**——这群人仍在中文
社交圈，**小红书确实能触达**。

**新 ICP 描述（待 Calibration 拍板）**：
> 海外华人专业人士（律师、医生、研究员等），需要在英文工作场景中
> 精准表达，但日常没有专门时间学英语。

**渠道判断（修正）**：小红书仍作为主线，**但内容切入点必须从"留学生/备考生"
转向"海外华人专业人士的工作语言痛点"**。

## 🚨 关键洞察：用户 A 案例揭示的真增长循环

```
小红书一篇笔记 → 用户 A 付费 → 深度交流 → 用户提 "某个核心功能" 建议
              → Qihang 大量迭代 → 产品进化
```

**这是典型 pre-PMF 增长循环**——内容是钓饵，真正价值在**和单个用户深聊**。
增长合伙人在这种状态下，工作重心是：

1. **少而精**的内容（一周 1-2 篇，每篇精准命中 ICP），不是 atu 当前每天产
2. **捕捉每个新进入用户**——评论 / DM / 注册后跟进
3. **系统化记录用户原话**（用户 A 提"某个核心功能"这类话永远不能忘——属于 Semantic Memory）
4. **帮 Qihang 准备和复盘 1-on-1 用户访谈**

这和 atu 当前的"批量内容供应链"工程形态**有显著差距**，是 Calibration 必须
拍板的最大决议。

## Round 2 答案

### 资源
- 审稿时间：**15-30 min/天**（Qihang 在大厂做 AI PM，工作时间长）
- 模型：愿用最强 Claude；atu 已是 Claude Code 驱动的
- 协作平台：**飞书 + lark-cli**（直接决定 Workbench backend = feishu）

### 渠道
- 主线：小红书（保留）
- 储备 SEO：知乎（保留，发产品功能介绍/同类产品评测/ICP 内容）
- 想试：LinkedIn / X 中文圈
- 策略修正：**Qihang 接受了"少而精"的方向**，不大量反复生产单一形式
- 不知道侦察/储备/暂缓还该有什么

### 红线
- 发布人审：**接受**
- AIGC %：Qihang 想从 20% 放宽到 5-10% — 🚨 待 Calibration 拍板
- 营销密度：和平台强相关，无固定数字
- 账号：**产品主号 + 个人账号**（不是矩阵号违规）
- 国内/海外社媒风控不同 — 重要，写进 Charter 红线

他在哪里跑

凡是你已经有 AI agent 的地方。

commando 不是一个新 AI 工具。它是一层，叠在你已经在用的任何 AI 工具上，让它从"聊天框"变成"真员工"。

1 — 装进 CLI agent

你住在终端里

commando 作为 CLI 工具装进系统。LLM 的活由你的 CLI agent 干，commando 给它配一份 Charter、一份 Schedule，和一组持久 Skill。

Claude Code Codex Kimi GLM Qwen

2 — 配上你的 IDE

你住在 IDE 里

用任何 AI 加持的 IDE 打开仓库，commando 的 Onboarding skill 在 chat 面板里加载，25 分钟产出你的数字搭档。

3 — 任何 CoWork 平台

你和团队在一起

任何有 base agent 在跑的平台都能用——包括国内外正在涌现的 cowork / devloop 平台。哪儿有 agent，commando 就叠在哪儿。

CoWork Claude Desktop ChatGPT Desktop 你自己的

凡是有 base agent 在跑的地方，commando 都能把它变成一个专属的数字搭档——有真实的岗位、真实的记忆，和一张在你飞书 / Notion 工作空间里的真实工位。

他多了什么

一份岗位、一本日历、一段记忆。

雇一个员工，三件事会发生：你告诉他岗位是什么、你把他排进日历、你给他一张存放工作的桌子。commando 给你的 AI 工具做的就是这三件事。

01 · 岗位说明书

他记得自己是谁

仓库里一份 markdown 写明这位搭档的岗位——服务什么产品、为谁工作、说话什么风格、什么红线不能碰。每次对话自动注入。他不会"忘记"。

文件：charter.md · 每次调用自动注入

02 · 一天的活

他干很多种事

不是一项技能——很多项。写笔记、复盘会议、扫竞品、整理用户反馈。像真员工一样，岗位上该干什么他就能干。你也可以随时给他派新活。

文件：skills/*.md · 引入 / 定制 / 生成

03 · 工作日历

他早上 8 点上岗

排班表写"早 8 点晨间简报，11 点稿件审核，周日 7 点周复盘"。操作系统调度器跑这张表——他自己干活，结果推到你飞书 / Notion。你不用记着要打开 ChatGPT。

文件：schedule.yaml · launchd / systemd / cron

为什么他的 Skill 不一样

通用 Skill 不懂你的业务。他的懂。

市面上 AI 工具的 Skill 大多是通用模板——"做一份 PPT"、"写一段代码"、"分析这份数据"。对谁都能用，对谁都不够好。commando 让你的搭档拥有真正懂你产品、你用户、你红线的 Skill。

"我的增长合伙人不写通用小红书帖。他用我的语气，写给我的ICP，套我的双语 bridge 公式。"

他获得贴身 Skill 的三条路径：

1. Onboarding 自动生成。25 分钟对话产出贴你产品、用户、渠道的专属 Skill。

2. 从社区一键引入。看到别人发的好 Skill？一个命令拉进来。

3. 自动按你的 Charter rebuild。引入的 Skill 不会留在 generic 模板状态——commando 会按你的 Charter 重写它的 prompt，让它说你的话、守你的红线。

01

Onboarding 产出定制 Skill

25 分钟对话 → skills/xhs-bilingual-bridge/ 已经知道你的 ICP 是"备考 IELTS 的中国大学生"。

02

引入一个社区 Skill

commando install @commando/competitive-watch — 从 Registry 拉一个通用的竞品扫描 Skill 进来。

03

按你的 Charter 自动 rebuild

commando 重写这个 Skill 的 prompt body。现在 competitive-watch 知道你的 ICP、你的红线、要追哪几家竞品。说你的话。

驱动方 Cursor + commando · 工件落在 Obsidian 知识库

为什么用 commando

加强你已经在付费的 AI 工具。

你已经在为 Claude $20/月、Cursor $20/月 付费。commando 不替代它们——而是让它们不再每次清空记忆、能按时上岗、真的接进你的工作空间。

①

跨 session 的持续记忆

Charter + Semantic memory 落在磁盘上。搭档记得你的产品、用户、上季度的反思。你随时从 Cursor 换到 Claude Code——记忆都跟你走。

②

住在你工作空间里，不是 chat tab 里

工件落在飞书多维表格、Notion 数据库、Obsidian 知识库——团队已经在用的地方。审稿卡片推到你 IM。你不用记着要打开 ChatGPT。

③

懂你产品的 Skill

Onboarding 产出贴你业务的 Skill。从社区引入的 Skill自动按你的 Charter rebuild。不再是"写一篇小红书"这种通用模板。

④

早上 8 点自己上岗

操作系统调度器——launchd、systemd、cron——跑你的搭档。晨间简报、周末复盘、月度税务检查。不需要 SaaS 守护进程，不订阅云服务。

开始使用

明早 8 点，你的搭档上岗。

30 秒装好。25 分钟 Onboarding。第 1 天：早上 8 点准时上岗，跑完一天 Schedule，飞书弹卡片找你。
